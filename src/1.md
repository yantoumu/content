## 🎯 **多API并发方案设计**

### **方案选择：负载均衡 + 故障转移**

我推荐采用**负载均衡策略**，原因如下：
1. **提高吞吐量**：多API同时工作，查询速度成倍提升
2. **降低单点压力**：分散请求负载
3. **增强可靠性**：一个API失败时自动转移到其他API
4. **资源最大化利用**：充分利用3个API的处理能力

def parallel_api_query(self, keyword_batches: Dict[int, List[str]]) -> Dict[str, Any]:
    """并发执行多API查询"""
    
    with concurrent.futures.ThreadPoolExecutor(max_workers=len(self.api_clients)) as executor:
        # 为每个API创建查询任务
        future_to_api = {
            executor.submit(self._query_api_batch, api_index, batch): api_index
            for api_index, batch in keyword_batches.items()
        }
        
        # 并发等待结果
        results = {}
        for future in concurrent.futures.as_completed(future_to_api):
            api_index = future_to_api[future]
            try:
                api_result = future.result(timeout=90)  # 总超时控制
                results.update(api_result)
            except Exception as e:
                # 故障转移处理
                self._handle_api_failure(api_index, future, keyword_batches)
```

class APIPerformanceMonitor:
    """API性能监控器"""
    
    def update_api_stats(self, api_index: int, response_time: float, success: bool):
        """更新API统计信息"""
        stats = self.api_stats[api_index]
        stats['total_requests'] += 1
        
        if success:
            stats['success_count'] += 1
            # 更新平均响应时间
            stats['avg_response_time'] = (
                stats['avg_response_time'] * (stats['success_count'] - 1) + response_time
            ) / stats['success_count']
        else:
            stats['fail_count'] += 1
            
        # 计算健康分数 (0-100)
        total_requests = stats['success_count'] + stats['fail_count']
        success_rate = stats['success_count'] / total_requests
        response_score = max(0, 100 - stats['avg_response_time'])  # 响应时间越短分数越高
        stats['health_score'] = (success_rate * 70 + response_score * 0.3)  # 成功率权重70%
```
